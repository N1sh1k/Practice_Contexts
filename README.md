## 1. Цель работы
Реализация кросс-браузерных автотестов для проверки корректности переключения между различными контекстами веб-страниц (вкладки браузера, вложенные фреймы и системные JavaScript-уведомления).

## 2. Технологический стек
* **Язык:** Python 3.13
* **Фреймворк:** Pytest
* **Инструмент автоматизации:** Selenium WebDriver
* **Отчетность:** Allure Framework
* **Архитектура:** Page Object Model (POM)

## 3. Описание реализованных тестов
1. **test_new_tab_opening:** Проверка открытия новой вкладки, переключения на неё и возврата в исходное состояние.
2. **test_nested_iframes_navigation:** Тестирование навигации в сложной структуре вложенных фреймов (Top -> Left -> Parent -> Middle -> Default -> Bottom).
3. **test_javascript_alert:** Взаимодействие с модальными окнами браузера (обнаружение, чтение текста, нажатие "OK").

## 4. Особенности реализации и решенные проблемы
* **Race Conditions (Состояние гонки):** При тестировании в Mozilla Firefox была выявлена задержка отрисовки контента в новой вкладке. Проблема решена внедрением **Explicit Waits** (явных ожиданий) через `WebDriverWait` и ожиданием видимости элемента (`visibility_of_element_located`).
* **Кросс-браузерность:** Настроена параметризация фикстур в `conftest.py`, позволяющая запускать весь набор тестов последовательно в Chrome и Firefox.
* **Логирование:** Внедрена система логирования действий (`logging`), которая фиксирует каждый шаг теста в консоли и отчетах.
* **Фиксация отклонений:** Реализован механизм автоматического снятия скриншотов при падении тестов (хук `pytest_runtest_makereport`).

## 5. Результаты
Тестирование успешно пройдено в обоих браузерах (6 тестов). Отчет сгенерирован в Allure. В ходе работы подтверждена стабильность фреймворка при работе со сложными контекстами.

## 6. Оптимизация и рефакторинг (День 6)

### Произведенные улучшения:
1. **Рефакторинг кода:** В `BasePage` внедрены комбинированные методы (например, `wait_and_switch_to_frame`), что позволило сократить объем кода в тестах на 25% и исключить дублирование логики Selenium.
2. **Оптимизация производительности:** Внедрен режим **Headless** для браузеров Chrome и Firefox. 
   * Время прогона 6 тестов в обычном режиме: **~120 сек.**
   * Время прогона в Headless-режиме: **82.79сек.**
   * Эффективность: ускорение процесса тестирования на **31%**.
3. **Фиксы Race Conditions:** 
   * Использованы динамические ожидания `EC.frame_to_be_available_and_switch_to_it` вместо жестких пауз.
   * Увеличено время тайм-аута до 15 секунд для обеспечения стабильности на медленных сетевых соединениях.

### Результат:
Достигнута 100% стабильность (3 прогона подряд без падений).

### Вывод по оптимизации:
Использование Headless-режима в сочетании с переходом на Explicit Waits (явные ожидания) позволило не только ускорить тесты, но и добиться 100% стабильности (6/6 passed) при кросс-браузерном запуске.
